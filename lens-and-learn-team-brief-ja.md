# Lens & Learn — チーム概要 (Team Brief)

*仮称。ハッカソンプロジェクト。開発時間：約4時間。*

## 概要 (The one-liner)

スマートフォンを被写体に向ける → 学習中のターゲット言語でそれらの単語を教えてくれる → 集めた単語が**1つの文章に統合され、AIによって画像が生成される**。これにより、視覚的に単語を記憶することができます。

## 機能（ユーザー視点）

1. **写真を撮影または選択する。** アプリは被写体を識別し、ターゲット言語でそれぞれにラベルを付けます（単語、発音表記（ピンインなど）、自然な例文、タップで発音を聞く機能）。
2. **単語をワードバンク（Word Bank）に保存する。** 記憶したい単語をタップします。
3. **結合（Forge）。** ボタンをタップすると、保存した単語が1つの自然な文章に編み込まれ、その文章を表すイラストが表示されます。散らばっていた語彙が1つの絵になって現れる様子を体験できます。

デモのデフォルトターゲット言語：**中国語（ピンイン付き）**。

## なぜ優れているのか

- **Google I/O 2026の最新発表技術**を採用：Gemini 3.5 Flash（高速マルチモーダルビジョン） ＋ Nano Banana 2 画像生成。
- **実証された学習科学**：単語と生成された画像をペアリングすることは「二重符号化（Dual Coding）」（言語メモリ ＋ 視覚メモリ）であり、単なる単語カードよりも記憶の定着が良くなります。
- **審査員に向けた責任あるAI（Responsible-AI）のアピール**：生成されるすべての画像には、不可視の **SynthID** 出所水印（ウォーターマーク）が含まれています。

## 仕組み（パイプライン）

```
写真 ──> Gemini 3.5 Flash (Vision) ──> ラベル付き単語（単語、ローマ字/ピンイン、例文）
                                               │
                          タップして保存 ───────┘
                                               ▼
ワードバンク ──> Gemini 3.5 Flash ──> { 文章 + 英語の画像生成プロンプト (image_prompt) }
                                               │
                            image_prompt ──────┘──> Nano Banana ──> イラスト
```

**重要なトリック：** 画像生成モデルには中国語（ターゲット言語）の文章を描くよう指示しません（文字の描画は不得意なため）。代わりに Gemini が文章と、その場面を描写したプレーンな英語の `image_prompt` を別々に生成します。画像モデルにはその英語のプロンプトのみを送信します。これにより、出力の信頼性が格段に向上します。

## 使用するGoogleテクノロジー（Google Cloud必須チェックリスト）

| 機能 | 用途 | モデル / API |
|---|---|---|
| Gemini 3.5 Flash | 写真 → 単語生成、および単語の結合 | Gemini API (`gemini-3.5-flash` — AI Studioで正確なIDを確認) |
| Nano Banana 2 | フレーズイラストの生成 | Gemini API (`gemini-3.1-flash-image`) |
| Cloud Text-to-Speech | 単語の発音 | `texttospeech.googleapis.com` (ボイス例: `cmn-CN-Wavenet-A`) |

3つすべてがシンプルな**APIキー認証**を使用しており、OAuthやサービスアカウントの設定は不要です。Google AI Studioから1つのキーを取得するだけです。

> 注意：Nano Banana 2 (`gemini-3.1-flash-image`) は課金の有効化が必要な場合があります。ゼロコストのフォールバックとして `gemini-2.5-flash-image`（初代Nano Banana、約500枚/日の無料枠）が使用可能です。**リクエストの形式は同一**で、モデル文字列を変更するだけです。

## 開発するもの（推奨分担）

- **メンバーA — キャプチャ ＋ 単語生成:** PhotosPicker → base64 → Gemini vision呼び出し（JSON出力を強制） → 単語カードのレンダリング。
- **メンバーB — 結合（Forge） ＋ 画像生成:** ワードバンク（メモリ内配列のみ、データベースなし） → 結合呼び出し → Nano Banana呼び出し → 結果画面（文章 ＋ 画像表示 ＋ ローディング状態）。
- **共通:** ネットワークヘルパーの作成 ＋ APIキーの共有。デモ用の魅力的な写真を1枚事前に用意。

スコープは絞る：ログイン機能なし、永続化なし、1言語のみ、美しいデモパス1つ。

## デモスクリプト（約75秒）

1. アプリを開き、部屋の画像を選択する。 → 単語が表示され、タップして発音を確認する。
2. 3〜4語を保存する。
3. **Forge**ボタンを押す → 文章が組み立てられ、イラストが描かれる。
4. 決め台詞：*"視覚化された文章で覚える、あなたのボキャブラリー。SynthIDによる出所証明付きです。"*
